reach 学习

介绍描述
1. 用于动态创建用户界面的javascript库(只关注视图)
2. 由Facebook开源
react的特点
1.声明式编码
2.组件化编码
3.React Native 编写原生应用
4.高效 (优秀的Diffing算法)
react高效的原因
1.使用虚拟机(virtual)DOM,不总是直接操作页面真实的DOM
2.DOM Diffing算法,最小化页面重绘
react依赖
1. babel.min.js es6转译es5 || jsxl转译js
2. react.development.js  react 核心库
3. react-dom.development.js 扩展操作dom的拓展库
4. prop-types.js
关于虚拟DMO
1. 本质是Object类型的对象(一般对象)
2. 虚拟DOM比较"轻",(轻指的是虚拟dom 和真实dom的元素比较，而虚拟dom比真实dom少很多元素),因为虚拟dom是React内部使用,无需真实dom上面那么多元素
3. 虚拟dom最终会被React转换为真实Dom,呈现在页面上
jsx语法
1. 定义虚拟Dom时，不要写引号
2. 标签中混入js表达式时要用{}
3. 样式的类名指定不要用class,要用className
4. 内联样式，要用style={{key:value}} 注意当key 是多个单词组成要用小驼峰的方式书写 例如font-size 得写fontSize
5. 虚拟dom必须只有一个根标签
6. 标签必须闭合 <input /> 或者<input></input>
7. 标签首字母
    (1)若小写字母开头,则将该标签转为html中同名元素,若html中无该标签对应的同名元素，则报错
    (2)若大写字母开头,react就会区渲染对应的组件，若组件没有定义,则报错
